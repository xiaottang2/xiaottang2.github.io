Tic Tac Toe AI using MDP and Q-learning

Tic Tac Toe is classic board game and the algorithmic approach of designing AI for it has been extensively studies, which usually requires building model according to the rules of game. It is desirable if we can design an AI without having the knowledge of the rule but by letting the agent learn how to play simply by trial and error.

This project addresses this issue by using model-free Q-learning. That is, the agent wonâ€™t be able to, or need to, understand the rule of the game. It masters the game by playing it for many times, usually more than 10 thousands of time, and then memorize the relationships between its moves and rewards.

As the total number (5478) of game states of TicTacToe is manageable, we can use a simple Q function - a Python dictionary. In a more complicated scence, for example, if we are dealing with Go, then we might need to use neural network to handle large number of states.

The following video shows the Q value of different possible actions in the board (i.e. where to put the â€˜Xâ€™ or â€˜Oâ€™). The agent always choose the highest Q value because Q value is a quantitative measure of how good the move is.

Note that at the initial stage, the Q value for the upper right cell is always the highest. This is not indicating that the upper right cell is better than all other cell, instead, this is due to the initial randomness when the agent is training. Remember when the agent make decisions, it either choose the action that has the highest Q value, or it choose a random action. The probability of either choice affects the training result, and thus explains why certain action is initially better than others. As a matter of fact, the four corners should share equal Q value as they are essentially symmetrical. In addition, if we take symmetry into account, the number of states of the game will reduce substantially and the training time will also decline.

Some questions remains:

  • How do we trade-off between exploration and exploitation?
  • How do we design the decay of discount rate?
